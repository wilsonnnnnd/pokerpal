# 统一状态管理系统使用指南

## 概述

已成功统一了项目中所有页面的状态管理，创建了类型安全的状态管理系统，包括加载状态、权限状态、错误状态、认证状态和分页状态。

## 状态类型定义

### 基础状态类型

```typescript
// 基础加载状态
interface LoadingState {
    loading: boolean;
    refreshing?: boolean;
    error?: string | null;
}

// 权限相关状态
interface PermissionState {
    isHost: boolean;
    permLoading: boolean;
    hasPermission?: boolean;
}

// 用户认证状态
interface AuthState {
    user: { /* 用户信息 */ } | null;
    persistedUser?: any | null;
    authLoading: boolean;
}

// 数据分页状态
interface PaginationState {
    hasNextPage: boolean;
    isLoadingMore: boolean;
    totalCount?: number;
    currentPage?: number;
}
```

### 组合状态类型

```typescript
// 通用页面状态
interface PageState extends LoadingState {
    initialized: boolean;
}

// 带权限的页面状态
interface PermissionPageState extends PageState, PermissionState {}

// 带认证的页面状态
interface AuthPageState extends PageState, AuthState {}

// 带分页的页面状态
interface PaginatedPageState extends PageState, PaginationState {}

// 完整的页面状态（包含所有状态类型）
interface FullPageState extends PageState, PermissionState, AuthState, PaginationState {}
```

## Hooks 使用方法

### 1. 基础页面状态 - usePageState()

```typescript
import { usePageState } from '@/hooks/usePageState';

function MyScreen() {
    const pageState = usePageState();
    
    // 可用的状态
    pageState.loading
    pageState.refreshing
    pageState.error
    pageState.initialized
    
    // 可用的操作
    pageState.setLoading(true)
    pageState.setRefreshing(true)
    pageState.setError('错误信息')
    pageState.setInitialized(true)
    pageState.reset()
}
```

### 2. 带权限的页面状态 - usePermissionPageState()

```typescript
import { usePermissionPageState } from '@/hooks/usePageState';

function HostOnlyScreen() {
    const pageState = usePermissionPageState();
    
    // 包含基础状态 + 权限状态
    pageState.loading
    pageState.isHost
    pageState.permLoading
    pageState.hasPermission
    
    pageState.setIsHost(true)
    pageState.setPermLoading(false)
}
```

### 3. 带分页的页面状态 - usePaginatedPageState()

```typescript
import { usePaginatedPageState } from '@/hooks/usePageState';

function ListScreen() {
    const pageState = usePaginatedPageState();
    
    // 包含基础状态 + 分页状态
    pageState.loading
    pageState.hasNextPage
    pageState.isLoadingMore
    pageState.totalCount
    
    pageState.setHasNextPage(false)
    pageState.setIsLoadingMore(true)
}
```

### 4. 完整状态 - useFullPageState()

```typescript
import { useFullPageState } from '@/hooks/usePageState';

function ComplexScreen() {
    const pageState = useFullPageState();
    
    // 包含所有状态类型
    // 基础状态、权限状态、认证状态、分页状态
}
```

## 优势

1. **类型安全**: 所有状态都有严格的 TypeScript 类型定义
2. **一致性**: 所有页面使用相同的状态管理模式
3. **可复用**: 状态组件可以在任何页面中复用
4. **易维护**: 状态逻辑集中管理，便于维护和更新
5. **减少重复**: 消除了各页面中重复的状态管理代码
6. **更好的UX**: 统一的加载、错误、空状态展示
